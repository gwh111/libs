# APP通用系统设计流程

  1. 定义模块。
    如 登录模块
  2. 罗列模块所有功能。
    如 找回密码、修改密码...
  3. 将功能分类。
    如 账户模块 = 充值 + 提现 + ...
  4. 根据分类建立模块结构，绘制模块核心架构图。
    模块层次尽可能纵向深入，平铺的要少。核心模块是给要自己开发UI的业务部门使用。
  5. 初步定义分类内每个功能的API。
    规范使用：方法 + 参数 （如多个参数第一个使用with）
    安卓：方法 + (参数1、参数2)
    库名：xxLib 方法：如 loginWithCell:xxx code:xxx
  6. 定义模块整体流程的起调API，包括UI。
    库名：xxKit 方法：如 presentLoginViewControllerWithMode:xxx
  7. 将模块架构图和定义的API和安卓端讨论并统一。
    统一API
  8. 和服务端对接接口调整API。服务端参与环节 测试参与环节
    评估统一的API是否足够完成功能需求。
  9. 评审通用的UI是否合理。 UI参与环节 测试参与环节
    确认是否剥离业务元素，是否有业务自己配置部分。
  10. 系分模块开发时间，给出交测时间。

总耗时 交测时间
  1. 绘制通用模块提供的UI。
    开发一套比较通用的UI，可直接使用的UI。写代码时间
  2. 接入核心模块。如多人开发可以和下一条并行
    在UI交互功能部分加入 xxLib 内提供的API。写代码时间
    如点击充值按钮后会调用 payLib 内 payWithxxx 方法。
  3. 实现核心模块功能。
    将 xxLib 内的方法实现。写代码时间
  4. 调试接入核心模块的UI。
    运行测试整个流程。
  5. 提交测试。测试参与环节
